# WeChat Layout Patterns / 公众号排版范式

This file distills repeatable WeChat article layout patterns into execution rules. Use it before writing or assembling the Word document.

## 一、基本判断

公众号排版的核心是手机单栏阅读。所有字号、段距、图片、引用框和分隔符都要服务一个问题:

读者在 390px 左右的手机宽度里,能不能顺着往下读。

## 二、六个稳定范式

### 1. 单栏窄正文

正文只走一条阅读线。不要做复杂左右分栏,不要把正文切成过多卡片。

执行规则:
- Word 内容宽 16cm;HTML 容器 `max-width:677px`。
- 粘贴到公众号后,正文两侧应保留自然留白;不要让文字贴边。
- 正文段落默认左对齐或两端对齐;不要大面积居中正文。

### 2. 16px 正文基准

公众号正文的舒适区通常落在 16px 左右。过小会累,过大会显得像海报。

执行规则:
- HTML 正文:16px。
- Word 正文:11pt,用于复制到公众号后接近 16px 视觉效果。
- 图注:12px/9pt 左右;引语:15px/10.5pt 左右;章节标题:21px/14pt 左右。
- 不为强调临时加字号;强调只用加粗或主题色。

### 3. 1.75 行距 + 段后距

公众号文章的呼吸主要来自行距和段间距。空行只在少数转场位置使用。

执行规则:
- HTML 正文 `line-height:2.05`,视觉接近公众号里 1.75–1.8 倍的舒适阅读区。
- Word 正文 line 380,after 260;段落之间靠段后距,少用空段。
- 普通段后距一致;章节前距、图注后距、引语后距才允许变化。
- 禁止连续空行堆留白。

### 4. 首屏三件套

首屏决定读者是否继续。首屏只放标题、头图、冷开场。

执行规则:
- 标题之后立即放头图或封面图。
- 头图后只放 1–2 段冷开场,不要放目录、作者介绍、过长摘要。
- 首屏第一段必须有场景、冲突或具体事实,不能从行业背景泛起。

### 5. 图前铺垫,图后收束

图片不能突然出现。每张图都要有读者理解它的入口和出口。

执行规则:
- 图前至少一段正文解释为什么看这张图。
- 图后至少一句正文收束图里的关键点。
- 图注只补充主体、来源、场景或判断,不重复前后正文。
- 连续两张图之间至少隔两段正文;除非是同一组并列图。

### 6. 模块少而稳

公众号长文最怕样式太多。稳定的少量模块比花哨模板更耐读。

允许模块:
- 正文段落
- 章节编号 + 章节标题
- 信息图/实景图 + 图注
- 引语块
- 分隔符
- 关于栏

限制:
- 全文 divider 不超过 2 处。
- 引语块 1–3 个。
- 每节加粗金句 1–2 个。
- 主题色强调全文不超过 4 处。
- 不使用复杂边框、浮层、动效、花字和装饰性图标。

## 三、文章长度与配图节奏

| 正文字数 | 建议图片数 | 节奏 |
|---|---:|---|
| 800–1200 | 1–3 | 头图 + 1 张解释图或场景图 |
| 1200–1800 | 2–4 | 头图 + 1–2 张信息图 + 1 张转场图 |
| 1800–2600 | 4–6 | 每个关键章节附近有图,但不连续堆图 |
| 2600+ | 按章节规划 | 每 700–900 字至少一次视觉缓冲 |

判断标准:
- 连续纯文字超过 6 段,需要处理。
- 连续短段超过 5 段,需要合并或加入解释段。
- 连续两屏都是图,需要删图或补正文。

## 四、封面和正文图

封面有两个任务:订阅号列表里能被看见,分享卡片里不被裁坏。

执行规则:
- 头条封面按 2.35:1 制作,建议 2400×1022 或 1200×511。
- 关键标题、Logo、人物脸、产品主体放在中心安全区,避免方形裁切时丢失。
- 正文横图宽度至少 1000px;信息图输出 2000px 宽 PNG。
- 单张图片尽量压缩到 2MB 内;微信上传上限更高,但加载速度会影响阅读。

## 五、四种常用版式

### A. 报道型

适合融资、公司故事、产品发布。

结构:
标题 → 头图 → 冷开场 2–4 段 → 01 事实冲突 → 02 主角选择 → 03 证据/客户/收入 → 04 去向 → 引语或关于栏。

排版重点:
- 首屏要快。
- 图片放在关键事实后。
- 引语用于补人味,不要替代事实。

### B. 拆解型

适合行业分析、方法论、产品机制。

结构:
标题 → 头图 → 本文要解决的问题 → 01 现状 → 02 机制 → 信息图 → 03 案例 → 04 可迁移结论 → 要点收束。

排版重点:
- 信息图跟着机制段走。
- 小标题用动词短语,不要写成目录名词。
- 结尾给读者可带走的 3–4 个判断。

### C. 访谈/人物型

适合创始人、投资人、技术负责人。

结构:
标题 → 人物/场景头图 → 一个动作或场景开场 → 章节推进 → 引语块 2–3 个 → 关于栏。

排版重点:
- 引语块少而准。
- 人物图不能像证件照;更需要场景感。
- 作者代拟引语必须在发布说明中标注待确认。

### D. 产品/技术型

适合产品升级、技术路线、解决方案。

结构:
标题 → 产品/场景头图 → 痛点场景 → 机制解释 → 信息图 → 使用案例 → 限制与下一步。

排版重点:
- 避免大段术语。
- 每个技术词后面给一个具体场景。
- 图中只放机制和公开指标,不放敏感竞品数据。

## 六、手机预览 QA

最终交付前,按 390px 宽度检查:

- 第一屏是否只出现标题、头图、冷开场。
- 任意一屏是否文字过密。
- 图前是否有铺垫,图后是否有收束。
- 小标题是否太长导致换行难看。
- 图注是否超过两行。
- 引语块是否过厚,影响继续阅读。
- 加粗是否太多,导致每段都像重点。
- 图片是否过大、过高或内容太小。

如果手机预览不顺,先改段落和图片位置,再考虑改文案。
