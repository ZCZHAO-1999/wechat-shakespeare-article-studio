# Editorial Guidelines / 编辑规范(口吻、结构、纪律)

This guide defines the voice, structure, title formulas, and numerical discipline for every article.

## 一、文章质量的底层目标

好文章先让人顺着读下去,再让人记住一个判断。深度、信息量、观点密度都要服务阅读体验。

- 第一屏:标题、头图、首段要给出明确场景或冲突,不铺背景材料。
- 每一屏:至少有一个推进点,可以是事实、判断、转折、引语或图。
- 每一节:只解决一个问题;小标题要可复述,不要写成目录标签。
- 每一个判断:贴着事实写,避免从抽象概念跳到抽象概念。
- 每一次加粗:承担停顿和记忆功能,不要把普通句子加粗成口号。

## 二、两个口吻档位

### A. 叙事体(财经科技报道腔)——默认档
适用:公司故事、产品发布、融资/里程碑、人物特写;读者为泛创投与从业者。
写法特征(全部执行):
- **短呼吸**:段落 1–3 句;一句话超过 38 个汉字时优先拆句。
- **冷开场**:三四段内抛出反差钩子(最热的赛道 × 最惨的收入;裁员新闻 × 人才缺口),
  并埋一个后文才收回的悬念("连做机器人的公司都找上了门")。
- **新闻腔缓冲**:据了解 / 据悉 / 据业内流传的数字 / 有业内人士指出——用于软化
  无法署名来源的判断与数字。
- **粗体打点**:每节 1–2 个加粗金句,承担"截图传播位";不要整段加粗。
- **引语块**:全文 1–3 个,放在团队/愿景/技术判断处;引语要有"人味 + 判断",
  不要客套话。结尾可用创始人引语收束。
- **节奏公式**:每节结尾要么一个钩子推向下一节,要么一记收束句。
- 禁:学术腔("综上所述")、堆砌形容词、连续排比超过三组、模板化二元转折。

### B. 拆解体(写给同行/创业团队的方法论)
适用:playbook、复盘、行业分析;读者为创业者/研究者。
- 保留叙事体的短段与粗体打点,但允许"判断句 + 论证段"的结构;
- 章节眉可用『0X + 动词短语』("引擎怎么自己变强");
- 结尾用编号要点(①②③④,每点=加粗论点+一句展开),再接互动问句。

档位判断:用户给了参照文→跟参照文;没给→看读者(泛读者选 A,同行选 B),拿不准
时选 A 并在交付说明里注明可切换。

## 三、结构库

标准骨架(可增删,顺序勿乱):
1. 头图 + 一行图注(品牌 · 系列名)
2. 冷开场 2–4 段(钩子 + 本文承诺:"这篇拆……,写给……")
3. 编号章节 01–06:
   - 冲突/行业事实(为什么现状不成立)
   - 主角的选择(一句话判断 + 机制展开)——机制段是全文技术含量所在,
     术语用作者原话,不自行发明词
   - 证据(收入/部署/客户;"收入即叙事"类小节)
   - 泛化/意外验证(第二曲线、跨界合作)
   - 去向(平行场景网格 + 唯一筛选标准)
   - 给读者的要点(拆解体)或愿景收束(叙事体)
4. 创始人/技术负责人引语(可选,置于对应章节或结尾)
5. 互动问句一句("你在哪条路线上?评论区聊聊。")
6. 「关于 XX」栏:公司一段话口径 + 官网,浅灰小字,上边框分隔

## 四、标题与摘要

标题公式(各写一个,选最锋利的当主标题):
- 反差数字型:『X 融了◻◻,年收入只有◻◻;它 40 天就超过了』
- 悬念判断型:『最热的赛道,最惨的收入:……』
- 身份宣言型:『别再叫它 EdTech / 别再卖记忆了:……』
- 机制命名型:『反馈即意图:拆解一台会自己变强的引擎』
约束:主标题 ≤ 64 字符;数字尽量放前半;不用感叹号。
摘要 ≤ 120 字:数字锚点 + 机制一句 + 读者定位一句。

## 五、AI 腔消毒

基础模型容易写出看似顺滑、实际空心的句子。成稿前逐段扫描,命中即改。

### 直接禁用的句法

- "不是 A,而是 B"、"与其说 A,不如说 B"、"真正的 X,从来不是 Y"。
- "本质上/归根结底/从某种意义上说/可以说/不得不说"开头的判断。
- "赋能、闭环、重塑、颠覆、破局、抓手、底座、范式、护城河"等空泛词,除非原稿已有且语境具体。
- 连续三句同构排比,例如"它让...它让...它让..."。
- 段尾万能收束,例如"这或许才是最大的想象空间"。

### 改写原则

- 用具体事实替代抽象词:把"重塑行业"改成"把交付周期从三周压到三天"。
- 用动作替代概念:把"形成闭环"改成"用户反馈会回写到下一轮推荐"。
- 用场景替代口号:把"赋能销售"改成"销售打开客户页时,先看到三条可追问线索"。
- 用平实判断替代二元转折:直接写"A 解决了什么问题,B 又留下什么代价"。
- 读不顺时先删半句,再重排顺序;不要补形容词。

## 六、数字与口径纪律(硬规则)

1. 建立**数字清单**:动笔前把草稿里所有数字/日期/专名抄成清单,写作时逐一对照;
   成稿后再全文扫一遍,任何清单外数字→删或占位『◻◻』并写进发布说明。
2. **竞品与敏感数字**:只出现在行文,前置缓冲语(据业内流传的数字/据了解),
   永不进信息图、封面、摘要、标题。理由:图与标题最易被截图脱离上下文传播。
3. **对标慎用**:用大公司做对标(如某招聘平台、某 SaaS)会把宏大叙事锚小;
   除非作者明确要求,默认不写"比 XX 早/强 N 年"式对标。
4. **平行原则**:多业务线默认等权呈现;"主线/主打"字样须作者点头。
5. **引语纪律**:代拟引语=基于作者提供的观点组织成口语,交付说明标注
   "代拟稿,发布前请本人确认";同一公众号跨文章不复用同一引语。
6. 涉及未公开融资/收入/合作,保留作者原话的模糊度,不擅自加精度。

## 七、成稿自检清单
- [ ] 口吻与参照/档位一致,无学术腔残留
- [ ] 无"不是 A,而是 B"式模板句;无空泛宏大词
- [ ] 手机阅读节奏顺:连续长段不超过 2 段,连续短句不超过 5 段
- [ ] 每节有钩子或收束;粗体金句每节 ≤2
- [ ] 数字全部来自清单;敏感数字带缓冲且不在图中
- [ ] 图注格式统一『品牌 · 描述』;FIG 编号连续
- [ ] 引语不与既往文章重复;代拟已标注
- [ ] 结尾互动句 + 关于栏齐备
